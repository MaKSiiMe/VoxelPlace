## ⚠️ Importance

La gestion de projet est **souvent négligée** par les candidats mais est **fondamentale pour le RNCP 6**.
C'est le **premier point à traiter** avant de commencer la conception et le développement.

---

## Concepts clés à maîtriser

- Gestion de projet
- Méthodes **agiles** (Scrum, Kanban, XP, Lean)
- Méthodes **non-agiles** (Waterfall, Cycle en V)
- Rôles Scrum : **Scrum Master**, **Product Owner**, équipe de dev
- **Product Backlog**
- **Sprint** (2 à 4 semaines)

---

## Méthodes agiles vs non-agiles

| | Agile | Non-agile (ex: Waterfall) |
|---|-------|--------------------------|
| Flexibilité | ✅ Haute | ❌ Faible |
| Livraisons | Itératives | Une seule à la fin |
| Client | Impliqué en continu | Peu impliqué |
| Documentation | Légère | Complète |
| Changements | Faciles à gérer | Difficiles |
| Inconvénient agile | Moins de documentation | Difficile à gérer les changements |

---

## Scrum — Rôles

### Product Owner
- Gère et **priorise le Product Backlog**
- Maximise la valeur du produit
- Représente les besoins du client

### Scrum Master
- **Facilite** les cérémonies Scrum
- **Supprime les obstacles** qui entravent l'équipe
- Garant de l'application de la méthode Scrum

### Équipe de développement
- Réalise les fonctionnalités
- Auto-organisée

---

## Scrum — Cérémonies

- **Sprint Planning** — planification du sprint
- **Daily Scrum** — réunion quotidienne (15 min)
- **Sprint Review** — démo du travail réalisé
- **Sprint Retrospective** — amélioration continue

---

## Méthode Waterfall

- Séquentielle : Analyse → Conception → Développement → Tests → Déploiement
- Chaque phase doit être terminée avant de passer à la suivante
- **Avantage** : documentation complète, vérification à chaque étape
- **Inconvénient** : difficile de gérer les changements en cours de projet

## Notes pour Claude Code

> Choisir et documenter une méthodologie dès le début du projet (Scrum recommandé).
> Créer un Product Backlog avec les features priorisées.
> Organiser le travail en sprints de 2 semaines.
> Documenter la méthodologie choisie dans le rapport avec les justifications.
> Utiliser GitHub Projects ou Trello pour visualiser l'avancement (tableau Kanban).
